How it works
01
Sign in.
Fund your agent.
Google login. A local wallet is generated. Put $20 in with your card via Privy onramp. Key never leaves your browser.
02
Pick a skill.
Skills are markdown files that tell the agent what it can do — which contracts, which commands, which strategy. Curated for GenLayer protocols.
03
Hit run.
A real Debian VM boots in your tab. The agent wakes up, reads the skill, starts executing. Real bash. Real cast. Real transactions.
04
Watch it work.
Filtered output shows what the agent is doing. Advanced users get a raw terminal — or even share it with the agent via tmux.
The stack
Runtime
SubZeroClaw
54KB C binary. Agentic loop, LLM calls, tool dispatch. Compiled for i386. Runs natively in the VM.
VM
CheerpX / WebVM
Real x86-32 Linux in the browser. Real bash, grep, awk, jq, pipes. No shims. No Node.js sandbox. Actual Debian.
Tools
Foundry cast (i686)
Cross-compiled for i386. Pure-compute commands run natively. Network calls bridge through browser fetch(). Transparent to skills.
Bridge
File-based IPC
curl writes request to /tmp/bridge/. Browser JS polls, does fetch(), writes response back. ~100ms overhead. Invisible.
Auth + Funding
Privy + viem
Google sign-in via Privy. Local wallet generated with viem. Card onramp funded directly to agent address. Key stays client-side.
Skills
Markdown files
Human-readable. Agent-readable. Environment spec, available commands, and workflow in one file. Write once, runs anywhere.
